119 /*
120  There is a typical latency from the timer interrupt until the first
121  actual line of code executes. Here is a typical compiler output of
122  approximately 34 cycles. When starting, this same delay is used to
123  begin counting, so the first reading will not have 34 cycles of
124  extra measurement. Because each measurement period gates instantly
125  after the previous, this approximate correction only affects the
126  first measurement. If you do not define TIMER_LATENCY_CYCLES, this
127  extra delay is skipped (saving a tiny bit of code space), and the
128  only downside is a slight inaccuracy in the first measurement.
129  2 00000326 <__vector_13>:
130  2 326: 1f 92 push r1
131  2 328: 0f 92 push r0
132  1 32a: 0f b6 in r0, 0x3f ; 63
133  2 32c: 0f 92 push r0
134  1 32e: 11 24 eor r1, r1
135  2 330: ef 92 push r14
136  2 332: ff 92 push r15
137  2 334: 0f 93 push r16
138  2 336: 1f 93 push r17
139  2 338: 2f 93 push r18
140  2 33a: 3f 93 push r19
141  2 33c: 4f 93 push r20
142  2 33e: 5f 93 push r21
143  2 340: 8f 93 push r24
144  2 342: 9f 93 push r25
145  2 344: af 93 push r26
146  2 346: bf 93 push r27
